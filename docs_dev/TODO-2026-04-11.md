## 2. Plomberie I/O & Serveur MCP (Tickets A2FP Nouveaux)
- [x] **[ISSUE-SEC-P1] Vulnérabilité DoS IPC :** Brider la variable de longueur de trame dans `ipc_pipeline.rs:40` (`MAX_PAYLOAD` 10MB) pour empêcher un attaquant de faire cracher un `vec!` arbitraire et OOM le système.
- [x] **[ISSUE-SEC-P2] Unbounded Line Reading Stdio :** Injecter une limite formelle au `next_line()` asynchrone sur `stdin` pour bloquer les tentatives de DoS par chaîne infinie.
- [x] **[ISSUE-PROTO-P2] Violation Stricte JSON-RPC 2.0 :** Refuser ou ignorer formellement les messages `stdio` dont le champ `id` est manquant sans émettre de réponse factice `id: null` (qui corrompt les Handshakes avec Claude/Cursor).
- [x] **[ISSUE-ARCH-P3] Faux-Semblant IPC :** Désenchevêtrer `handle_request()` de `stdio_pipeline.rs` pour l'importer dans un routeur agnostique dans `lib.rs` (C'est fait, `execute_domain` mis en place et l'IPC supporte désormais tout).
- [x] **[ISSUE-PERF-P3] Zero-Allocation IPC :** Maintenir un pool tampon permanent (`BytesMut` ou `Vec` réutilisable `.clear()`) pour purger la lourde instanciation malloc perpétuelle lors du parsing Rkyv.
- [x] **[ISSUE-PERF-CPU] Anti-Starvation Rkyv :** Isolé via `tokio::task::block_in_place` (Audit de RustyMaster) empêchant toute Starvation sans rompre la Zero-Allocation (Pas de `'static` lifetime et donc pas de Clone inutile imposé).
- [x] **[ISSUE-ARCH-DUAL] Unification Dual-Plane MCP :** Adopter l'architecture Acteur "Dual-Plane" recommandée par RustArch : le Hub ne dépend plus du transport (Stdio ou Socket). L'IPC doit parser le handshake via le même Routeur centralisé.
