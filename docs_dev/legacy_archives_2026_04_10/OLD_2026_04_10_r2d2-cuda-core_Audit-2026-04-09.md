# Audit R2D2 - Sovereign Shield (r2d2-cuda-core)

**Date :** 2026-04-09
**Expert Sollicité :** Cuda
**Status Global :** FFI basique. Refonte totale requise pour atteindre le niveau Industriel Zéro-Crash.

## Retour d'Experise

La crate Cuda actuelle contourne le GPU via un simple fallback CPU injecté avec `extern "C"`. L'Expert Kernel a formellement rejeté l'approche des FFI C traditionnelles pour un environnement de Haute Assurance :

1. **Linker Host Instable :** La compilation des headers Cuda sur système hôte (Fedora 42) crée un fort taux de rejet. Il faut substituer l'édition de lien statique par la fonctionnalité "Dynamic Loading" de `cudarc`.
2. **Insécurité des Pointeurs Bruts :** L'actuelle allocation utilise `*const u8`, violant l'Ownership Rust et favorisant les Segfaults silencieux. `cudarc` impose le `CudaSlice<T>`.
3. **Panic & VRAM :** Les retours de kernels doivent obligatoirement utiliser les API "Safe" de `cudarc` s'interfaçant avec le système type `Result::Err` pour ne jamais faire paniquer l'Orchestrateur asynchrone en cas de Memory-Bound ou d'Arguments Kernels invalides.
4. **Engorgement Malloc :** L'emploi à venir des CudaMalloc standards entraînera des goulots d'étranglement avec nos micro-batches. Une transition vers `cudaMallocAsync` est réclamée.
